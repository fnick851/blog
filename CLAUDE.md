# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

Personal blog (noah-song.com) generated by its own ~750-line static site generator — deliberately no framework. It replaced Hexo in July 2026; the generator's output was verified byte-equivalent to the final Hexo build, and much of its behavior exists to preserve that compatibility.

## Commands

```bash
npm run build              # generate site into public/ (~80 ms)
npm test                   # build + smoke-test the invariants below (node:test)
npm run serve              # http://localhost:4000, rebuilds on any change
npm run new "Post Title"   # scaffold source/_posts/YYYY-MM-DD-Post-Title.md
```

Run `npm test` after any generator change — its assertions encode the invariants below and were derived from the Hexo-parity verification. There is no linter.

`serve.mjs` re-imports `build.mjs` with a cache-busting query on each change, so edits to the generator itself take effect without restarting the server.

## Architecture

Pipeline: `source/_posts/*.md` → frontmatter parsed in `build.mjs` → markdown rendered by `lib/md.mjs` → wrapped by `lib/templates.mjs` → written to `public/`, plus atom.xml, sitemap.xml/txt; `assets/` and `source/img/` are copied verbatim. Site-wide constants (title, links, timezone, feed size) live in `site.config.mjs`.

Dependency policy: exactly `marked`, `highlight.js`, `js-yaml`. A minimal flat dependency tree is the point of this codebase — do not add dependencies for things a few lines of Node can do.

## Invariants — breaking these breaks real things

- **URLs are frozen**: `/:year/:month/:day/:slug/`, slug = post filename minus date prefix, verbatim (CJK characters included; percent-encode only in emitted hrefs/feeds, as `lib/md.mjs#encodeURL` does).
- **Heading ids are Hexo-style** (case-preserving, `-` separators, via `slugize`) with `<a class="headerlink">` anchors. Inbound deep links like `#The-null-Type` depend on this. Don't switch to marked's default lowercase slugs.
- **The stylesheet must stay render-blocking** (`<link rel="stylesheet">` in `lib/templates.mjs`). An async/preload CSS hack caused the site's historical flash-of-unstyled-content bug. Do not "optimize" this, even if Lighthouse suggests it.
- **HTML structure and class names in `lib/templates.mjs`, `lib/toc.mjs`, and `lib/md.mjs` must match `assets/css/style.css`** (`.post-wrap`, `.archive-item`, `figure.highlight` table/gutter markup, `.toc-list`/`.is-active-link`, `.iconfont icon-<name>`, etc.). The CSS is hand-authored plain CSS (design tokens in `:root`, responsive overrides nested inside each component's section); there is no CSS build step. It was rewritten from the old theme's compiled output in July 2026 and verified equivalent by computed-style diffing, preserving the theme's rendered quirks (e.g. hljs `.number` tokens are green, not the Tomorrow theme's orange). The social icons come from a font embedded as a data URI — keep the full glyph set so config-driven links can be re-enabled.
- **Post dates are wall-clock America/New_York** strings (`YYYY-MM-DD HH:mm:ss`). `build.mjs` converts them to UTC instants for the feed. Frontmatter is parsed with js-yaml `CORE_SCHEMA` specifically so dates stay strings — `DEFAULT_SCHEMA` would parse them as UTC Dates and shift every timestamp.
- Markdown rendering keeps Hexo semantics: `breaks: true` (single newline → `<br>`), smartypants quotes applied only outside raw HTML/URLs, raw HTML passes through untouched (posts rely on this — one embeds a `<script>`), fenced code renders as hexo-util's line-numbered `<figure class="highlight">` table.
- External links get `target="_blank" rel="noopener"` injected by `externalLinks()` in `lib/md.mjs`, which runs on every full page at write time — templates and markdown don't add these attributes themselves.

## Content model

Posts have exactly `title` and `date` frontmatter — no tags, categories, or drafts, and the generator supports nothing else. The atom feed includes only the newest `feedLimit` (20) posts and sets `updated` = published date by design.

## Deployment

Vercel builds on push to `master`: `vercel.json` pins `npm run build` + `public/` (overriding any stale dashboard settings), and `engines: >=20` in package.json selects Vercel's newest Node. Pushing a non-master branch produces a preview deployment.
